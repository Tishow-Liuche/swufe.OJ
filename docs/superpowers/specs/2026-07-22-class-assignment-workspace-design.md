# 班级与作业选题工作台设计

## 目标

让教师无需进入“查看全部班级”即可切换教学班，并在发布作业时使用与题库一致的筛选和批量选题流程。发布页需同时展示已选题单，便于核对与移除。

## 已确认方案

采用“题库表格优先 + 常驻题单”的组合方案。

- 当前班级区直接列出教师拥有的班级；展开侧栏时以可滚动列表展示全部班级，当前班级高亮。窄屏改为横向可滚动的班级项。
- 发布作业页包含标题、截止时间和说明，以及题库式筛选表格。
- 题目筛选支持关键词、来源、算法标签和难度同时生效。
- 表格支持单题勾选、取消勾选和全选/取消全选当前页结果。
- 桌面端右侧常驻“作业题单”；窄屏移至题目表格下方。题单显示题目数、已选题目及单题移除操作。

## 交互与状态

### 班级选择

班级列表使用现有 `classes` 和 `selectedClassId` 状态。点击任一班级后调用现有 `loadClassData`，刷新成员和作业数据，并保持当前工作面板不变。侧栏收起时保留紧凑的当前班级切换按钮，避免占用窄栏空间。

### 题库筛选与批量选题

发布页新增以下本地状态：关键词、来源、标签、难度、页码、题目总数、筛选元数据和加载状态。进入发布页时加载 `/api/problems/metadata`；筛选或翻页时请求现有 `/api/problems`，传递 `keyword`、`source`、`tag`、`difficulty`、`page`、`pageSize` 和 `status=PUBLISHED`。

`selectedProblems` 继续作为跨页的作业题单。每个表格行的复选框根据该数组判断状态；表头复选框只操作当前结果页。筛选改变不会清空已经加入题单的题目，教师可通过右侧题单逐项移除或一键清空。

### 发布

继续调用 `POST /api/teacher/assignments`，提交当前 `selectedClassId`、作业基础信息和题单中的 `problemIds`。不改变作业、题目或班级的数据模型，也不新增后端接口。

## 页面结构

发布页由三部分组成：

1. 作业基本信息表单与已选数量。
2. 左侧/主区域题库表格：筛选栏、题目行、分页与当前页批量操作。
3. 右侧题单：选题数、题目名称、来源/标签/难度摘要、移除和清空；底部放置发布操作。

桌面端采用主表格加约 260px 题单栏的两列布局；宽度不足时变为单列，题单紧随表格，确保文本和按钮不会溢出。

## 异常与空状态

- 题库元数据或题目列表加载失败时，使用现有消息栏提示失败原因，并保留上一次可用的题单。
- 筛选无结果时，主表格显示空状态，不影响已选题单。
- 未选择班级、未填写标题或题单为空时，沿用现有发布校验。
- 班级列表为空时，显示创建班级的现有引导，不渲染无效的发布入口。

## 验证

- 前端类型检查和生产构建。
- 后端现有完整测试套件，确保作业发布接口无回归。
- 手动验证：班级直接切换、四种筛选组合、单题与当前页批量选题、跨页保留题单、移除/清空、发布成功与窄屏布局。

## 范围外

- 不支持“跨所有筛选结果一键全选”，避免在未知总量下创建隐式题单。
- 不新增题目筛选维度或修改题库、作业和班级的持久化模型。
